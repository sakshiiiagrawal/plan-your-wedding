# TypeScript Migration & Architecture Refactor Plan

## Context
WeddingPlanner is a multi-tenant wedding planning SaaS (React frontend + Express backend on Vercel + Supabase). The entire codebase is plain JavaScript/JSX with no types. The goal is a full TypeScript migration with a feature-based modular architecture, zero hardcodings, open-source readiness, and no performance regressions.

## Key Architectural Decisions

### 1. Shared Types вҖ” No Monorepo Complexity
`shared/` is consumed **via TypeScript path aliases only** вҖ” no npm workspaces, no publishing, no `npm link`. Both `api/tsconfig.json` and `frontend/tsconfig.json` map `@wedding-planner/shared` вҶ’ `../shared/src/index.ts`. Zero runtime coupling; pure compile-time sharing.

### 2. Supabase as Type Source of Truth
Run `supabase gen types typescript` вҶ’ `shared/src/supabase.generated.ts`. All domain types **derive** from generated types вҖ” never duplicate:
```typescript
// shared/src/domain/guest.types.ts
import type { Database } from '../supabase.generated';
export type GuestRow = Database['public']['Tables']['guests']['Row'];
export type GuestInsert = Database['public']['Tables']['guests']['Insert'];
export interface GuestWithDetails extends GuestRow {
  guest_groups: { name: string } | null;
  room_allocations: RoomAllocationWithRoom[];
}
```

### 3. Enums вҖ” `as const` Arrays, Not TypeScript `enum`
```typescript
// shared/src/enums/guest.enums.ts
export const GUEST_SIDES = ['bride', 'groom', 'mutual'] as const;
export type GuestSide = typeof GUEST_SIDES[number]; // 'bride' | 'groom' | 'mutual'
```
Same value usable at runtime (for validation, `Array.includes()`) and as a compile-time type union.

### 4. Backend: Controller вҶ’ Service вҶ’ Repository
```
Route вҶ’ Controller (HTTP layer) вҶ’ Service (business logic) вҶ’ Repository (Supabase queries)
```
This makes each layer independently readable and testable. The budget module (currently 680-line controller) is the primary motivation.

### 5. Typed `req.user` via Declaration Merging
```typescript
// api/_src/types/express.d.ts
declare global {
  namespace Express {
    interface Request { user?: AuthenticatedUser; }
  }
}
```
`getAuthUser(req)` throws `UnauthorizedError` if `req.user` is undefined вҖ” no `!` assertions anywhere.

### 6. Error Hierarchy
```typescript
AppError (base) вҶ’ HttpError variants:
  NotFoundError (404), UnauthorizedError (401),
  ForbiddenError (403), ConflictError (409), ValidationError (400)
```
Error middleware checks `instanceof AppError` first, then maps Supabase errors, then returns 500.

### 7. Zod for All Validation (replaces Joi + custom `validateRequiredFields`)
- Backend: Zod schemas in each module's `*.validator.ts`
- Frontend: same schemas (imported from `shared/`) for form validation via `react-hook-form`'s `zodResolver`
- Env config: Zod schemas in `api/_src/config/env.ts` and `frontend/src/config/env.ts` вҖ” fail fast on startup if env is invalid

### 8. `tsx` Runtime for Vercel (no `dist/` compilation)
Vercel's `@vercel/node` runtime transpiles TypeScript via esbuild natively. `api/index.ts` вҶ’ serverless function. Type checking is done separately by `tsc --noEmit` in CI вҖ” not as part of deployment. Eliminates the `dist/` artifact problem entirely.

### 9. Frontend Hooks вҖ” Module-Scoped, Not Monolithic
`useApi.js` (600 lines) is decomposed into one hooks file per domain:
```typescript
// src/modules/guests/hooks/useGuests.ts
export const GUEST_QUERY_KEYS = {
  all: ['guests'] as const,
  list: (filters: GuestFilters) => ['guests', 'list', filters] as const,
  detail: (id: string) => ['guests', id] as const,
} as const;

export const useGuests = (filters: GuestFilters = {}) =>
  useQuery<GuestWithDetails[]>({
    queryKey: GUEST_QUERY_KEYS.list(filters),
    queryFn: () => api.get('/guests', { params: filters }).then(r => r.data),
  });
```
`useApi.js` remains functional during migration; pages switch imports one by one until it can be deleted.

### 10. Typed Route Constants
```typescript
// src/routes/routes.ts
export const ROUTES = {
  home: '/',
  onboard: '/onboard',
  login: (slug: string) => `/${slug}/login`,
  admin: {
    guests: (slug: string) => `/${slug}/admin/guests`,
    budget: (slug: string) => `/${slug}/admin/budget`,
    // ...
  },
} as const;
```

## Environment Configuration

### `api/_src/config/env.ts`
```typescript
import { z } from 'zod';
const envSchema = z.object({
  NODE_ENV: z.enum(['development', 'production', 'test']).default('development'),
  PORT: z.coerce.number().default(3001),
  JWT_SECRET: z.string().min(32, 'JWT_SECRET must be at least 32 characters'),
  SUPABASE_URL: z.string().url(),
  SUPABASE_SERVICE_KEY: z.string().min(1),
  FRONTEND_URL: z.string().url().optional(),
});
const parsed = envSchema.safeParse(process.env);
if (!parsed.success) { console.error(parsed.error.flatten()); process.exit(1); }
export const env = parsed.data;
```

## Migration Phases

### Phase 0: Infrastructure (no feature changes)
- Install: `typescript`, `@types/node`, `@types/express`, `tsx`, `zod`
- Create `shared/` with tsconfig + generate Supabase types
- Create `api/tsconfig.json`, `frontend/tsconfig.json`, `vite.config.ts` with path aliases
- Create error class hierarchy (`AppError`, `HttpError` variants)
- Migrate `env.ts` with Zod validation (replaces `constants.js` + inline `process.env`)
- Add `express.d.ts` declaration merge for `req.user`
- Set up ESLint, Prettier, Husky
- Create `scripts/setup.sh` and `.env.example`

### Phase 1: Auth Module (foundation for everything else)
- `shared/src/domain/user.types.ts` + `shared/src/api/auth.dto.ts`
- `auth.middleware.ts` (typed `req.user`)
- `modules/auth/` (controller + service + validator + routes)
- `modules/users/`, `modules/setup/`
- Frontend: `AuthContext.tsx` with typed `AuthenticatedUser`

### Phase 2: Core Event Domains (dependency order)
Migrate one complete domain at a time: **events вҶ’ venues вҶ’ guests вҶ’ accommodations**
For each: shared types вҶ’ repository вҶ’ service вҶ’ controller вҶ’ validator вҶ’ routes вҶ’ frontend hooks вҶ’ frontend page

### Phase 3: Financial Domains
**vendors вҶ’ budget вҶ’ tasks**
Budget is the most complex (18 methods, 680 lines) вҖ” the repository pattern extracts ~15 Supabase queries out of the controller.

### Phase 4: Infrastructure Domains
**dashboard вҶ’ website-content вҶ’ weddings**

### Phase 5: Frontend Completion
- Convert remaining `.jsx` вҶ’ `.tsx` (layouts, Marketing, Onboard steps)
- Replace all magic route strings with `ROUTES` constants
- Delete `useApi.js` once all pages have switched to module hooks
- Add `React.lazy()` for route-level code splitting

### Phase 6: Open Source Polish
- `README.md` (features, stack, quick start, architecture diagram)
- `CONTRIBUTING.md` (adding a module guide, TypeScript conventions, commit format)
- `LICENSE` (MIT)
- GitHub issue templates + PR template
- Audit for any hardcoded personal names (server.js has "Sakshi & Ayush")
